# スプリント3 検収記録(記録者: ミナト / 2026-07-19)

## 発注者フィードバック(起点)
「右側パネルが画面を圧迫して編集マス目が見えにくい。普段は閉じておいて、ボタンで開閉できるように」

**体制としての分析(バグゼロ運用)**: スプリント2のボトムシート化は640px以下にしか効いておらず、それより広い画面(PCや大きめの表示)では従来どおり右パネルが常時250px占有していた。「どの画面幅で使われるか」の想定確認が不足していた。→ 対策: 目視検証はモバイル幅とデスクトップ幅の**両方**で行うことをチェックリストに追加。

## 実装(サクラ)
- **全画面幅でサイドバー(道具箱)は既定で閉**。
- 641px以上: 右からスライドするドロワー(#appBody基準のoverlay、幅280px)。ヘッダーの「🧰 道具箱」ボタンで開閉、開いている間はボタンが強調表示。
- 640px以下: 既存のボトムシート(取っ手でもヘッダーボタンでも開閉可、開閉ロジックはtoggleSheetに共通化)。
- 編集マス目はどの幅でも全幅を使える。

## 途中で発見・修正した不具合(正直な記録)
- 初回実装でドロワーがヘッダーまで覆う不具合を**目視検証で発見**(サイドバーのDOM上の親を#editBodyと誤認 → 実際は#appBody直下。位置基準を修正)。ロジックテストでは検出できない見た目の事故であり、DoD2(実物確認)が機能した実例。
