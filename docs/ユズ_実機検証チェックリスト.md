# iPhone実機検証チェックリスト(担当: ユズ / 毎リリース実施)

> **運用ルール**: 1つでも☐のまま(=落ちた)なら、そのリリースは止める(ユズの拒否権)。
> 検証結果はこのファイルをコピーして「検証記録_日付.md」として残す。
> ユーザーがバグを発見した場合は体制の失敗として扱い、すり抜けた穴をこのリストに**必ず追加**する。

## 0. 前提
- [ ] 検証環境をメモ: iPhoneの機種 / iOSバージョン / Safariかホーム画面起動か
- [ ] `npm test` 全項目グリーンを確認してから実機に進む(グリーンでないものは実機に持ち込まない)

## 1. 表示の3面一致(最重要 — 過去事故の再発防止)
- [ ] 編集画面・実機風プレビュー・**実機Spoonの表示**の3つで形が一致する
- [ ] 点字アートが**ななめに崩れない**(アート行のスペース内訳が行ごとに揃っているか)
- [ ] 幅ロックが**コピー結果に効いている**(1行目がロック幅+見えないアンカーで終わる)

## 2. コピー → Spoon貼り付け(コア動線)
- [ ] コピーボタン→Spoonのプロフィールコメント欄に貼り付け→**形が崩れない**
- [ ] 1行目の行頭が消えない(点字空白アンカーが効いている)
- [ ] 空行が消えない(ハングルフィラーが効いている)
- [ ] 文字数カウントとSpoon側の受け入れが矛盾しない(200字制限)
- [ ] 📱 点字空白(U+2800)・ハングルフィラー(U+FFA0)がクリップボード経由で欠落しない
- [ ] コピー失敗時に手動コピー退避モーダルが出て、そこからの手動コピーでも形が崩れない

## 3. タッチ操作
- [ ] タップで要素を選択できる(サイドシートに選択情報が出る)
- [ ] ドラッグで要素を移動できる(**ページスクロールに奪われない**)
- [ ] 長押し0.5秒で複数選択に追加/解除できる(iOSの文字選択メニューが出ない)
- [ ] 複数選択のままドラッグ/◀▲▼▶で一緒に動く
- [ ] ◀▲▼▶ボタンで1マスずつ動く(半角許可行では半マス)
- [ ] ダブルタップで編集(テキスト)/ドット編集(アート)が開く ※iOSのdblclick挙動要確認
- [ ] ボトムシート(道具箱)の開閉がスムーズ/シートを開いたまま編集マス目が操作できる
- [ ] ピンチズームで編集マス目を拡大できる(未実装なら次スプリントへ)

## 4. ファイルI/O
- [ ] 📱 保存: 保存ボタン→ファイルAppに .song.json が保存される
- [ ] 📱 読込: 読込ボタン→ファイルAppから .song.json を選択できる(acceptで弾かれないか)
- [ ] 保存→読込の往復でレイアウト・まとまり・回転設定が完全に戻る
- [ ] 📱 画像→点字: 写真ライブラリから取り込める / HEIC写真でも動く

## 5. PWA
- [ ] 「ホーム画面に追加」でアイコンとアプリ名(定期スタジオ)が正しく出る
- [ ] 📱 ホーム画面から起動するとフルスクリーン(standalone)で開く
- [ ] 📱 機内モードで起動しても動く(Service Workerキャッシュ)
- [ ] 縦向きで全機能が操作できる(375px相当)

## 6. フル曲モード(タッチ)
- [ ] フィルムストリップの横スクロールとページ切替がタッチでできる
- [ ] 境界の◀▶移動・行グリップ(●)のドラッグがタッチでできる
- [ ] コピー→次ページ自動送り→…→最後のページまでの一連の流れが完走できる
